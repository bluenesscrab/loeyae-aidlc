# UI Mock 工作流（两阶段编排 + iframe 拆分）

核心思想：**先确认页面流程对不对，再往单个页面填充内容**。用户审两次：一次审流程骨架，一次审最终效果。

**架构模式**：每个页面单独实现为独立 HTML 文件，主文件通过 iframe 引入，避免单端 HTML 文件过大。

---

## 流程总览

```
用户故事 + 需求文档
       ↓
【阶段1：页面编排 + 骨架】→ 产出 page-specs.md + 主文件骨架 + 子页面骨架
       ↓
用户审核：页面数量对吗？流程跳转对吗？改造的页面对吗？
       ↓  确认
【阶段2：内容填充】→ 填充子页面 UI 内容 + 主文件条件表/说明
       ↓
用户审核：每个页面的字段、条件表、交互对吗？
```

---

## 文件结构与职责分离

### 目录结构

```
{模块}/inception/ui-mock/
├── {端}-page-specs.md            # 页面清单（阶段1产出）
├── platform-admin.html           # 主文件（iframe 汇总）
├── platform-admin/               # 子页面文件夹，与主文件同名
│   ├── rule-list.html
│   ├── rule-edit.html
│   └── ...
├── user-app.html
└── user-app/
    ├── home.html
    └── ...
```

**命名规则**：
- 主文件名 = 端标识（如 `platform-admin.html`）
- 子文件夹名 = 主文件名去掉 `.html`
- 子页面文件名 = 页面英文短名（kebab-case）

### 子页面 HTML

**只包含**：纯 UI 视觉内容
- 完整独立的 HTML
- 样式全部内联
- **不包含**：标题栏、条件表、推导说明、业务规则

### 主文件 HTML

**包含**：
1. 页面头部（用户角色、核心任务声明）
2. iframe 缩放脚本
3. 每个页面的 mock-box 容器

---

## 阶段1：页面编排 + 骨架

### 目标

快速确定：有哪些页面、每个页面是改造还是新增、页面间怎么跳转。

### 产出

1. `{端}-page-specs.md` — 精简清单表 + 操作闭环
2. `{端}.html` — 主文件骨架
3. `{端}/xxx.html` — 子页面骨架

### page-specs.md 格式

```markdown
## 页面清单表
| # | mock-box 名称 | 子页面文件名 | 类型 | 关联 US | 改造基础 |
|---|---|---|---|---|---|
| 1 | 提交订单 | submit-order.html | 局部改动 | US-03 | pages/order/index.vue |
| 2 | 卡券详情 | ticket-detail.html | 新增页面 | US-14 | — |

## 操作闭环检查
| 按钮/入口 | 目标 | 状态 |
|---|---|---|
| [确认核销] | → #3 核销结果页 | ✅ |
```

### 骨架内容规则

- **改造页面**：完整还原现有页面结构（读取代码后 1:1 还原），不做改动标注
- **新增页面**：只画页面标题 + 操作按钮列表

### 用户审核目标（阶段1）

- ✅ 页面数量对不对？
- ✅ 改造的是不是正确的页面？
- ✅ 页面间的跳转关系对不对？
- ✅ 有没有操作闭环问题？

---

## 阶段2：内容填充

### 前提

用户已确认阶段1的骨架。

### 执行方式

逐个页面填充完整内容：
- **改造页面**：在已还原结构上，用橙色边框标注增量改动
- **新增页面**：基于骨架中确认的按钮列表，填入完整 UI 内容

### 填充时的强制检查

- [ ] 改造页面：去掉橙色标记后是否和现有系统高度一致？
- [ ] 新增页面：骨架中每个按钮是否都有对应交互实现？
- [ ] 条件表是否覆盖了所有状态差异？
- [ ] page-specs.md 中的子页面文件名与实际文件一致？

---

## 异常处理

- "漏了某个页面" → 补到清单 + 创建骨架
- "这个页面不需要改" → 从清单移除
- "跳转关系不对" → 修改按钮标注
- 阶段2发现骨架有误 → 先回退修正骨架再继续
