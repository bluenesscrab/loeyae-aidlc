## 深度决策

**选择深度**：全面。

**理由**：

1. 用户会在任意受支持客户端直接经历恢复、审批交接、局部阻断、离线降级和故障恢复。
2. state v2 到 v3 迁移与 Legacy 零调用要求存在高回归风险。
3. 共享协议核心、三平台薄适配和 SSOT Provider 构成当前两个产品仓库的跨团队协作，但未来业务项目只维护自身单一工作区流程。
4. 平台中立、分发隔离、契约固定、事件游标和真实证据均需要可测试的异常场景。
5. I13 需要从故事 Gherkin 场景派生迁移、兼容、故障和端到端接力测试。

## 预期产出

- **画像数量**：5 个角色画像，覆盖需求中全部利益相关者。
- **故事粒度**：功能级、可独立验收的垂直切片；共享核心与客户端适配通过业务结果而非实现任务表达。
- **拆分方式建议**：以用户旅程为主、业务领域为辅；每个故事回链 `ADLC-FR-001`—`ADLC-FR-016` 和相关 NFR，并仅通过消费契约引用 SSOT 产品依赖。
- **拓扑边界**：当前双仓用于 SSOT 与 AI-DLC 产品研发；后续业务项目只维护自身一套 AI-DLC 状态和过程产物。
- **验收标准**：每个故事至少包含 happy path 与关键 error path；存在可选路径时补充 alternate path；全部使用 Given/When/Then。
- **强制产物**：`user-stories/personas.md`、`user-stories/stories.md`。
