# 构建和测试

**目的**：构建所有单元并执行全面的测试策略

## 前置条件
- 所有单元的代码生成必须完成
- 所有代码产物必须已生成
- 项目已准备好进行构建和测试

---

## 步骤 1：分析测试需求

分析项目以确定适当的测试策略：
- **单元测试**：在代码生成期间已按单元生成
- **集成测试**：测试单元/服务间的交互
- **性能测试**：负载、压力和可扩展性测试
- **端到端测试**：完整用户工作流
- **契约测试**：服务间 API 契约验证
- **安全测试**：漏洞扫描、渗透测试

---
