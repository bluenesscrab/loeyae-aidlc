# Session 交接提示词

## 目的

在阶段/步骤完成时，生成可复制的提示词并写入 state.md，方便：
- 用户在新 session 中继续工作（避免单 session 过长导致 token 浪费）
- 团队协作时接手人直接从 state.md 获取交接信息

## 机制

### 1. 完成消息中展示

每个阶段的完成消息模板已内置 `📋 新 Session 继续` 选项（无需额外加载本文件）。

### 2. 写入 state.md

**每个阶段完成时，必须同步更新 `state.md` 顶部的"下一步交接"字段。**

写入位置：`state.md` 的 `## 下一步交接` section（紧跟 `## 项目信息` 之后）。

### 3. 提示词生成规则

根据当前实际进度动态填充：

```markdown
## 下一步交接

> 复制以下提示词到新对话即可继续：

```
使用 AI-DLC，继续 {项目/模块名} 的开发。

当前状态：
- 阶段：{INCEPTION/CONSTRUCTION}
- 已完成：{上一步骤名}
- 下一步：{下一步骤名}
- 架构模式：{单模块/多模块}
{多模块时} - 活跃模块：{模块名}
{团队模式时} - 协作模式：{接力/认领}

请读取 state.md 恢复上下文，从「{下一步骤名}」开始。
```
```

### 4. 特殊场景

#### Construction 单元完成 → 下一单元

```
使用 AI-DLC，继续 {项目名} 的 Construction 阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成单元：{unit-name}（代码生成+审查通过）
- 下一单元：{next-unit-name}
- 单元进度：{已完成 X/Y 个单元}

请读取 state.md 恢复上下文，开始单元「{next-unit-name}」的开发。
```

#### 所有单元完成 → 构建和测试

```
使用 AI-DLC，继续 {项目名} 的最终阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成：所有单元代码生成和审查
- 下一步：最终全局审查 + 构建和测试

请读取 state.md 恢复上下文，执行最终全局审查和构建测试。
```

#### 产品级 Inception 完成 → 模块开发

```
使用 AI-DLC，继续 {项目名} 的模块开发。

当前状态：
- 阶段：模块选择
- 架构模式：多模块
- 已完成：产品级 Inception（模块划分 + 接口契约）
- 可用模块：{模块列表}

请读取 state.md 恢复上下文，展示模块选择菜单。
```

## 注意事项

1. **提示词必须自包含** — 新 session 的 AI 仅凭提示词 + state.md 就能恢复工作
2. **不要在提示词中包含大段上下文** — state.md 和 decision-summary 已经记录了关键信息
3. **保持简洁** — 提示词控制在 5-8 行以内
4. **确保 state.md 已更新** — 生成提示词前，当前步骤的完成状态必须已写入 state.md
5. **团队协作场景** — 接手人无需依赖上一人的聊天记录，直接查看 state.md 即可获取交接提示词
