# Session 交接提示词

## 目的

在阶段/步骤完成时，生成可复制的提示词并写入 state.md，方便：
- 用户在新 session 中继续工作（避免单 session 过长导致 token 浪费）
- 团队协作时接手人直接从 state.md 获取交接信息

## 机制

### 1. 完成消息中展示

每个阶段的完成消息模板已内置 `📋 新 Session 继续` 选项（无需额外加载本文件）。

### 2. 写入 state.md

**每个阶段完成时，必须同步更新 `state.md` 的"下一步交接"表格。**

写入位置：`state.md` 的 `## 下一步交接` section（紧跟 `# AI-DLC 状态跟踪` 标题之后，`## 项目信息` 之前）。

**写入规则**：
- **单人模式**：表格中只维护一行（范围 = 项目名）
- **团队协作 + 单模块模式**：按当前操作人的角色/单元维护行（追加或更新对应行）
- **多模块模式**：按模块维护行（每个模块一行），另有一行"产品级"
- **更新而非覆盖**：如果表格中已有对应范围的行，更新该行；否则追加新行

### 3. state.md 中的格式

```markdown
## 下一步交接

| 范围 | 更新时间 | 提示词 |
|------|----------|--------|
| {范围名} | {ISO日期} | `使用 AI-DLC，继续 {描述}...` |
```

**范围命名规则**：
- 单人模式：`项目名`
- 产品级 Inception：`产品级`
- 模块级：`{module-name} 模块`
- 单元级：`{unit-name} 单元`

### 4. 提示词生成规则

根据当前实际进度动态填充提示词内容：

#### 通用格式

```
使用 AI-DLC，继续 {项目/模块名} 的开发。

当前状态：
- 阶段：{INCEPTION/CONSTRUCTION}
- 已完成：{上一步骤名}
- 下一步：{下一步骤名}
- 架构模式：{单模块/多模块}
{多模块时} - 活跃模块：{模块名}
{团队模式时} - 协作模式：{接力/认领}

请读取 state.md 恢复上下文，从「{下一步骤名}」开始。
```

#### Construction 单元完成 → 下一单元

```
使用 AI-DLC，继续 {项目名} 的 Construction 阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成单元：{unit-name}（代码生成+审查通过）
- 下一单元：{next-unit-name}
- 单元进度：{已完成 X/Y 个单元}

请读取 state.md 恢复上下文，开始单元「{next-unit-name}」的开发。
```

#### 所有单元完成 → 构建和测试

```
使用 AI-DLC，继续 {项目名} 的最终阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成：所有单元代码生成和审查
- 下一步：最终全局审查 + 构建和测试

请读取 state.md 恢复上下文，执行最终全局审查和构建测试。
```

#### 产品级 Inception 完成 → 模块开发

```
使用 AI-DLC，继续 {项目名} 的模块开发。

当前状态：
- 阶段：模块选择
- 架构模式：多模块
- 已完成：产品级 Inception（模块划分 + 接口契约）
- 可用模块：{模块列表}

请读取 state.md 恢复上下文，展示模块选择菜单。
```

### 5. 示例

#### 单人模式

```markdown
## 下一步交接

| 范围 | 更新时间 | 提示词 |
|------|----------|--------|
| 订单管理系统 | 2025-01-15 | `使用 AI-DLC，继续订单管理系统的 Construction。已完成：order-service 单元，下一步：order-query 单元。请读取 state.md 恢复上下文。` |
```

#### 多模块 + 团队协作

```markdown
## 下一步交接

| 范围 | 更新时间 | 提示词 |
|------|----------|--------|
| 产品级 | 2025-01-15 | `使用 AI-DLC，继续模块开发。产品级 Inception 已完成，请读取 state.md 展示模块选择菜单。` |
| base 模块 | 2025-01-16 | `使用 AI-DLC，继续 base 模块的 Construction。已完成：security-service 单元，下一步：common-service 单元。请读取 state.md 恢复上下文。` |
| order 模块 | 2025-01-16 | `使用 AI-DLC，继续 order 模块的 Inception。已完成：需求分析，下一步：用户故事。请读取 state.md 恢复上下文。` |
```

## 注意事项

1. **提示词必须自包含** — 新 session 的 AI 仅凭提示词 + state.md 就能恢复工作
2. **不要在提示词中包含大段上下文** — state.md 和 decision-summary 已经记录了关键信息
3. **保持简洁** — 表格中的提示词控制在 1-2 行以内（用户复制后可展开使用）
4. **确保 state.md 已更新** — 生成提示词前，当前步骤的完成状态必须已写入 state.md
5. **团队协作场景** — 接手人无需依赖上一人的聊天记录，直接查看 state.md 即可获取交接提示词
6. **Git 冲突最小化** — 各人/模块各自维护自己的行，追加行而非修改同一行，降低合并冲突概率
