# Session 交接提示词

## 目的

在阶段/步骤完成时，除了提供"继续"选项外，额外生成一个可复制的提示词，用户可以在新 session 中粘贴使用，避免单个 session 过长导致 token 浪费。

## 触发时机

在以下节点的完成消息中，**必须**附带新 session 提示词：

1. **Inception 各步骤完成时** — 工作区检测、逆向工程、需求分析、用户故事、工作流规划、应用设计、单元生成
2. **Construction 各单元完成时** — 每个单元的代码生成+审查通过后
3. **产品级 Inception 完成时** — 模块划分和契约定义完成后
4. **模块级阶段转换时** — 模块 Inception → Construction

## 提示词格式

### 固定框架

```markdown
📋 **新 Session 提示词**（复制到新对话即可继续）：

---
使用 AI-DLC，继续 {项目/模块名} 的开发。

当前状态：
- 阶段：{INCEPTION/CONSTRUCTION}
- 已完成：{上一步骤名}
- 下一步：{下一步骤名}
- 架构模式：{单模块/多模块}
{多模块时} - 活跃模块：{模块名}
{团队模式时} - 协作模式：{接力/认领}

请读取 state.md 恢复上下文，从「{下一步骤名}」开始。
---
```

### 动态填充规则

AI 必须根据当前实际进度填充以下字段：

| 字段 | 来源 |
|------|------|
| 项目/模块名 | state.md 中的项目名或活跃模块名 |
| 阶段 | 当前所处的大阶段 |
| 已完成 | 刚刚完成的步骤名称 |
| 下一步 | 根据工作流逻辑确定的下一步骤 |
| 架构模式 | state.md 中记录的模式 |
| 活跃模块 | 仅多模块模式，state.md 中的活跃模块 |
| 协作模式 | 仅团队模式，state.md 中的协作模式 |

### 特殊场景的提示词

#### Construction 单元完成 → 下一单元

```markdown
使用 AI-DLC，继续 {项目名} 的 Construction 阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成单元：{unit-name}（代码生成+审查通过）
- 下一单元：{next-unit-name}
- 单元进度：{已完成 X/Y 个单元}

请读取 state.md 恢复上下文，开始单元「{next-unit-name}」的开发。
```

#### 所有单元完成 → 构建和测试

```markdown
使用 AI-DLC，继续 {项目名} 的最终阶段。

当前状态：
- 阶段：CONSTRUCTION
- 已完成：所有单元代码生成和审查
- 下一步：最终全局审查 + 构建和测试

请读取 state.md 恢复上下文，执行最终全局审查和构建测试。
```

#### 产品级 Inception 完成 → 模块开发

```markdown
使用 AI-DLC，继续 {项目名} 的模块开发。

当前状态：
- 阶段：模块选择
- 架构模式：多模块
- 已完成：产品级 Inception（模块划分 + 接口契约）
- 可用模块：{模块列表}

请读取 state.md 恢复上下文，展示模块选择菜单。
```

## 完成消息中的集成方式

在现有完成消息的选项之后，追加提示词块：

```markdown
> **🚀 <u>**下一步？**</u>**
>
> **你可以：**
>
> 🔧 **请求修改** - 根据审查结果要求修改
> ✅ **继续下一阶段** - 确认并进入**{下一阶段名称}**
> 📋 **新 Session 继续** - 复制以下提示词到新对话：

---

```
{动态生成的提示词}
```

---
```

## 注意事项

1. **提示词必须自包含** — 新 session 的 AI 仅凭提示词 + state.md 就能恢复工作
2. **不要在提示词中包含大段上下文** — state.md 和 decision-summary 已经记录了关键信息
3. **保持简洁** — 提示词控制在 5-8 行以内
4. **确保 state.md 已更新** — 生成提示词前，当前步骤的完成状态必须已写入 state.md
