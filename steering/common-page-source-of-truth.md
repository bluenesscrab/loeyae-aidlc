# 页面级单一真相源规则

## 核心问题

AI-DLC 产出物按**功能/单元**维度组织，但前端页面可能被多个单元共享。当多个单元的产出物都描述了同一个页面的部分内容时，后续修改（修 bug、调整功能）可能只参照其中一份产出物，导致其他单元贡献的功能被意外移除。

## 核心规则

**每个前端页面在同一时间只能有一份权威的完整描述**。当多个单元涉及同一个页面时，该页面的最终状态必须合并到一个统一的位置。

## 权威来源定义

| 产出物类型 | 页面权威来源 | 说明 |
|-----------|------------|------|
| UI Mock（标准模式） | 对应端的 HTML 文件中的该页面 mock-box | 一个页面只有一个 mock-box |
| UI Mock（大型模式） | 对应端目录下对应模块 HTML 文件中的该页面 mock-box | 按模块定位文件，再定位 mock-box |
| 需求文档 | 该页面所属原始单元的需求文件 | 后续迁入的功能必须追加到此文件 |
| 用户故事 | 涉及该页面的所有故事，但需标注来源单元 | 通过页面名称可检索到完整故事集 |

**大型模式下的定位规则**：当 UI Mock 按模块拆分为多个文件时，定位某页面的 mock-box 需要先确定该页面属于哪个模块文件。定位方法：
1. 查看对应端目录的 `index.html`，根据页面名称确定所在模块
2. 打开对应模块 HTML 文件，搜索页面名称定位 mock-box

## 判定标准：何时触发回写

以下任何一种情况发生时，必须执行基准产出物回写（参见 `change-request-process.md` CR4.1）：

1. **功能迁入**：将其他模块/单元的功能移入某个已有页面
2. **入口新增**：在已有页面上新增功能入口（按钮、菜单项、Tab 等）
3. **布局调整**：对已有页面的区块布局进行增删改
4. **功能移除**：从已有页面移除某功能入口

## 回写规则

### UI Mock 回写

- 直接修改对应端 HTML 文件中该页面的 `mock-box`
- 禁止为同一页面创建第二个 `mock-box`
- 如果是"局部改动"，使用橙色高亮标记新增部分

### 需求文档回写

- 定位该页面在需求文档中的描述段落
- 将新功能追加到该段落中
- 如果页面跨单元，在追加内容后标注来源：`[来源: {单元名} 变更]`

### 用户故事回写

- 新增故事条目到该页面相关的故事文件中
- 故事标题中标注关联页面，便于检索

## 后续修改时的上下文加载

当修改某个页面（修 bug、调整样式、添加功能）时，必须按页面维度聚合上下文，不能仅加载某一个单元的产出物。

详细加载策略参见 `common-session-continuity.md` 的「页面级修改上下文加载」部分。

## 违规示例

- ❌ AI 绘图迁入 C 端 AI 中心页面，但 AI 中心的 UI Mock 和需求文档未更新 → 后续修 bug 时 AI 参照旧 UI Mock 删除了 AI 绘图入口
- ❌ 同一页面在单元 A 的需求中描述为"3 个 Tab"，在单元 B 的变更产出物中描述为"4 个 Tab" → 两份矛盾描述共存
- ❌ 变更完成后，新功能的需求只记录在变更日志中，未回写到页面基准产出物

## 合规示例

- ✅ AI 绘图迁入后，直接修改 `user-app.html` 中 AI 中心的 mock-box，增加 AI 绘图入口
- ✅ 需求文档中 AI 中心的描述更新为包含 AI 绘图功能的完整版本
- ✅ 管理后台产出物中标记"AI 绘图已迁移至 C 端 AI 中心，本处不再维护"

## 事件来源

此规则源于实际案例：边界重构将功能从管理后台迁移到 C 端，但 C 端页面的基准产出物（UI Mock、需求文档）未同步更新。后续修 bug 时 AI 助手参照未更新的旧版产出物，将迁入的功能入口删除。
