# 步骤完成强制协议（Step Completion Protocol）

> **为什么需要这个协议**：state.md 是三个工具入口（Kiro/Claude Code/OpenCode）恢复上下文的唯一依据。不更新 state.md = 下一次 session 丢失进度 = 重复工作。这不是仪式，是团队协作的基础设施。

**每个步骤完成时，必须执行以下协议，无例外，包括快速通道。**

---

## 协议内容（3 项，顺序不可调换）

```
步骤完成 → ① 更新 state.md → ② 输出微型摘要 → ③ 记录审计
```

### ① 更新 state.md（强制，不可跳过）

更新以下字段（根据当前步骤类型选择适用项）：
- `当前步骤` — 更新为刚完成的步骤名
- `阶段进度` — 对应步骤标记 ✅ 完成
- `下一步交接` — 更新提示词（参见 `common-session-handoff.md`）
- `质量门禁状态` — 如果执行了门禁检查，记录结果

**最小更新格式**（快速通道也必须执行）：
```markdown
## 阶段进度
| 步骤 | 状态 | 完成时间 |
|------|------|----------|
| {步骤名} | ✅ 完成 | {ISO时间} |
```

### ② 输出微型摘要（强制，嵌入完成消息中）

每个步骤的完成消息**末尾**必须包含以下格式的状态块：

```markdown
---
📍 **进度更新**
- 已完成：{步骤名}
- 下一步：{下一步骤名}
- state.md：✅ 已更新
```

**为什么在消息中输出**：这是自我验证机制。如果你无法填写"state.md：✅ 已更新"，说明你忘了更新。

### ③ 记录审计（强制）

在对应的分段审计文件中追加记录（格式见 `common-audit-logging.md`）。

---

## 违反协议的后果

如果在下一步骤开始时检测到 state.md 未更新（当前步骤字段与实际进度不符）：
1. **立即暂停**当前步骤
2. 补充更新 state.md
3. 在审计中标记"补充更新（协议违规恢复）"
4. 然后继续

---

## 快速通道的简化协议

快速通道任务虽然流程精简，但仍必须在以下节点更新 state.md：
- 工作区检测完成后
- 代码生成完成后
- 构建验证完成后

---

## 计划级复选框执行

### 计划执行的强制规则
1. **不得在未更新计划复选框的情况下完成任何工作**
2. **完成计划文件中描述的任何步骤后，立即标记该步骤 [x]**
3. **必须在完成工作的同一交互中完成**
4. **无例外**：每个计划步骤的完成都必须通过复选框更新跟踪

### 两级复选框跟踪系统
- **计划级**：跟踪每个阶段内的详细执行进度
- **阶段级**：在 state.md 中跟踪整体工作流进度
- **立即更新**：所有进度更新在完成工作的同一交互中完成
