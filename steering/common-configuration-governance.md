# 配置治理

**职责**：识别运行时配置的定义、发布范围、消费者、变更风险和验证证据。具体配置中心、文件格式和发布工具由条件技术适配或项目规范决定。

## 执行条件

存在环境配置、远程配置、特性开关、动态路由、限流规则、外部端点或多消费者共享配置时执行。仅编译期常量且不影响部署环境时跳过。

## 安全边界

- 不读取、复制或写入 Secret 的真实值；只记录键名、来源类别、Owner 和注入方式。
- 不在 AI-DLC 文档保存令牌、密码、证书私钥或完整敏感日志。
- 未经用户明确授权，不执行配置发布、环境切换或生产变更。

## 配置清单

在 `<system-baseline-root>/configuration-inventory.md` 记录：

| 字段 | 要求 |
|------|------|
| `config_id` | 项目内稳定标识，不暴露敏感值 |
| 权威来源 | 文件、配置平台或部署参数的位置 |
| 环境与作用域 | 环境、租户、区域、服务或实例范围 |
| 消费者 | 服务/组件反向索引及读取证据 |
| 数据类别 | 普通、敏感引用、动态规则或特性开关 |
| 默认值语义 | 缺失时行为；未知时标记待确认 |
| 生效方式 | 启动时、动态刷新、重启或重新部署 |
| Owner | 维护者和审批责任人 |

不复制完整配置正文；仅保留影响分析所需的键、结构摘要和证据引用。

## 变更风险

配置变更满足任一条件时不得走快速通道：
- 多个运行时消费者；
- 动态刷新或无法保证原子生效；
- 改变路由、鉴权、限流、降级、数据源或外部端点；
- 涉及 Secret 引用、环境继承或默认值语义；
- 新旧版本并存期间读取方式不同。

## 执行流程

1. 定位配置权威来源、作用域和生效机制。
2. 从代码、部署描述或平台导出信息建立消费者反向索引。
3. 比较键、类型、默认值、约束、环境覆盖和刷新行为。
4. 识别同时生效、分批生效和回滚时的版本组合。
5. 生成消费者适配与验证矩阵，纳入 CR、设计或 C8。
6. 更新清单的新鲜度、变更证据和未验证项。

## 验证矩阵

至少覆盖适用项：
- 配置结构或语法校验；
- 每个消费者的绑定/解析测试；
- 缺失、非法值和旧版本配置行为；
- 动态刷新或重启后的生效验证；
- 新旧应用版本与新旧配置的兼容组合；
- 回滚后行为与敏感信息不泄露检查。

无法访问配置环境时，记录本地可执行检查和外部 CI/平台证据要求，状态保持“未验证”。

## 完成标准

- [ ] 受影响配置有权威来源、作用域和 Owner
- [ ] 所有已知消费者有读取证据和适配状态
- [ ] 生效、版本组合与回滚行为明确
- [ ] 敏感值未进入 AI-DLC 产物
- [ ] 验证矩阵有实际命令或可信外部证据
- [ ] `state.md` 记录受影响配置、清单新鲜度和阻断项
