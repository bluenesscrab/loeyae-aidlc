# 模块边界约束规则

## 核心规则

执行任何批量操作（审计、自检、交叉验证、批量修复）时，**必须严格限定在 state.md 中声明的「活跃模块」范围内**。

## 判定标准

1. 在执行审计/自检前，先从 state.md 读取「活跃模块」字段
2. 确定当前操作的文件范围：`docs/aidlc/modules/{活跃模块}/` 下的文件
3. **禁止**将其他模块的文件纳入当前审计/自检/修复范围
4. 如果需要跨模块操作，必须先向用户确认

## 常见违规场景

- ❌ 审计联票 UI Mock 时，顺带审计了子账号的 HTML 文件
- ❌ 修复联票问题时，顺带修复了其他模块的类似问题
- ✅ 审计范围严格限定在当前活跃模块目录下
- ✅ 发现其他模块有类似问题时，仅记录到 state.md 待办，不主动修复

## 执行前自检

每次执行批量操作前，默念：

1. 当前活跃模块是什么？→ 只操作这个模块的文件
2. 我即将操作的文件列表中，有没有不属于当前模块的？→ 有则剔除
3. 发现了其他模块的问题怎么办？→ 记录，不修复
