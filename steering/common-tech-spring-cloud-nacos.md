# Spring Cloud / Nacos 条件技术适配

**职责**：把 Spring Cloud 与 Nacos 项目中的可验证技术事实映射到 AI-DLC 的通用服务、运行时依赖、契约和配置模型。本文不定义生命周期步骤，不替代项目规范，也不要求使用 Maven、Feign 或特定组件。

## 加载与跳过条件

仅在代码或配置中发现以下任一可靠证据时加载：
- Spring Cloud 相关依赖、插件或配置；
- Nacos 服务发现或配置客户端依赖；
- 项目已有配置明确声明对应组件。

仅凭目录名、用户口头提及但工作区无证据时，先记录“待确认”，不得自动套用。未检测到对应技术时跳过本文件。

在 `state.md` 记录：
- `技术适配: Spring Cloud/Nacos`
- 检测证据路径及版本来源；
- 实际存在的能力：服务发现、远程配置、网关、声明式客户端、消息等；
- 未检测或未验证的能力。

禁止把“使用 Spring Cloud”推断为所有组件均已启用。

## 通用模型映射

| 项目事实 | 映射目标 | 说明 |
|----------|----------|------|
| 应用名与部署配置 | 服务目录中的可部署单元候选 | 必须结合仓库与部署证据确认，不能只看配置名 |
| 服务发现注册信息 | 运行时服务节点与调用目标 | 环境、分组、命名空间等只记录项目实际值的脱敏引用 |
| HTTP/声明式客户端 | `sync-api` 运行时依赖 | 同时查找实际 URL、服务标识、回退和消费者 |
| 网关路由 | 外部入口与下游反向索引 | 动态来源未知时标记未验证 |
| 消息生产者/消费者 | `async-message` 依赖 | 主题、消费组、事件 Schema 以项目事实为准 |
| 远程配置引用 | `shared-config` 依赖 | 建立配置项到消费者的反向索引 |
| 数据访问配置 | 数据 Owner 候选 | 多服务共享数据源必须标记风险，不据此认可共享写入 |

## 证据提取

按项目实际构建系统和结构查找，不预设 Maven：
1. 构建清单中的 Spring Cloud、Nacos 和实际客户端依赖；
2. 应用配置、环境配置、启动参数和部署描述中的应用标识与远程配置引用；
3. 客户端声明、HTTP 调用、服务发现调用、网关路由和消息绑定；
4. 配置绑定类、动态刷新点和默认值；
5. 项目已有测试、CI 脚本和配置校验命令。

若项目使用 Maven，可读取 `pom.xml` 与 wrapper；若使用 Gradle，则读取对应构建脚本与 wrapper；其他构建方式沿用项目事实。禁止臆造 `mvn`、`gradle` 或任何脚本命令。

## Nacos 配置处理

- Data ID、Group、Namespace、环境映射和发布权限属于业务项目事实，只写入该项目的 `<system-baseline-root>/configuration-inventory.md`，不得写入本共享规则。
- 不读取或记录配置值中的 Secret；仅保留键名、来源和敏感引用方式。
- 必须区分本地默认值、启动阶段配置、运行阶段配置和动态刷新行为。
- 配置消费者从配置引用、绑定类和部署参数交叉确认；只发现配置项但未找到消费者时标记待确认。
- 无法访问 Nacos 服务端时，不得声称已验证线上配置；可保留代码侧和导出物侧证据。

## 服务调用与契约

- 客户端接口不是契约的唯一事实来源；优先引用项目已有 OpenAPI、Schema、生成源或提供方接口定义。
- 发现客户端与提供方定义不一致时，记录为契约冲突并阻断“已验证”。
- 回退、超时、重试、负载均衡和熔断能力必须以实际配置/代码为证据，不因框架存在而假设启用。
- 网关、客户端和直接 HTTP 调用都必须进入消费者反向索引。

## 验证命令发现

从 wrapper、构建清单、CI 和项目脚本中提取实际命令，并按受影响服务分别记录：
- 编译与单元测试；
- 契约或集成测试；
- 配置绑定/启动验证；
- 网关路由或消息绑定验证；
- 多服务联合验证。

需要 Nacos、注册中心或外部环境而当前不可用时，状态为“未验证”，并记录用户可在目标环境执行的项目真实命令或 CI 任务标识。

## 禁止行为

- 不创建或修改真实 Nacos 配置，不执行生产发布；
- 不假设所有服务都使用同一 Namespace、Group、版本或构建工具；
- 不把 Java Entity 作为跨服务共享契约；
- 不因存在依赖就宣称服务发现、配置刷新、熔断或消息能力有效；
- 不把本文件内容复制到 core、Skill 或平台入口。

## 完成标准

- [ ] 加载条件有工作区证据，技术能力逐项记录而非整体假设
- [ ] 服务、调用、消息和配置已映射到通用基线
- [ ] 构建与验证命令来自项目真实入口
- [ ] 客户端、提供方和机器契约之间的冲突已显式记录
- [ ] Nacos 环境事实与敏感值未写入共享规则或日志
- [ ] 无法访问的环境能力保持“未验证”
